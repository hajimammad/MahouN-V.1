4. FORTRESS VALIDATOR IMPLEMENTATION
================================================================================

LOCATION:
/home/haji/Desktop/MahouN/mahoun/core/fortress_validator.py

LINES OF CODE: ~1000+ (production-grade)

CORE CLASSES:

4.1 FortressValidator
---------------------
Main validation engine that intercepts and validates all reasoning responses.

INITIALIZATION:
    validator = FortressValidator(
        config_path=Path("constitution/RedLines.yaml"),
        execution_mode=ExecutionMode.DESKTOP_MINIMAL,
        strict_mode=True
    )

PARAMETERS:
- config_path: Path to RedLines.yaml (auto-detected if None)
- execution_mode: DESKTOP_MINIMAL or ENTERPRISE_FULL
- strict_mode: If True, raise exceptions; if False, log only

MAIN METHOD:
    async def validate(
        response: ReasoningResponse,
        correlation_id: Optional[str] = None
    ) -> ValidationResult

RETURNS:
- ValidationResult with pass/fail status
- Forensic context (hash, timestamp, violations)
- Statistics update

RAISES:
- SecurityBreachException on critical violations (if strict_mode=True)

4.2 SecurityBreachException
---------------------------
Non-bypassable exception raised on RedLine violations.

ATTRIBUTES:
- message: Human-readable violation description
- violation_type: ViolationType enum (MISSING_PROOF_TREE, LOW_AGREEMENT_SCORE, etc.)
- severity: ViolationSeverity enum (CRITICAL, HIGH, MEDIUM, LOW)
- forensic_context: Dict with full violation details
- correlation_id: Request tracing ID
- timestamp: ISO 8601 timestamp

EXAMPLE:
    SecurityBreachException(
        message="Agreement score 0.65 below threshold 0.85",
        violation_type=ViolationType.LOW_AGREEMENT_SCORE,
        severity=ViolationSeverity.CRITICAL,
        forensic_context={...},
        correlation_id="req-12345"
    )

4.3 ValidationResult
--------------------
Pydantic model containing validation outcome.

FIELDS:
- passed: bool (True if all checks passed)
- correlation_id: str (tracing ID)
- timestamp: str (ISO 8601)
- execution_time_ms: float (validation duration)
- violations: List[Dict] (all detected violations)
- warnings: List[str] (non-critical issues)
- forensic_hash: str (SHA256 of response, 16 chars)
- metadata: Dict (execution mode, checks performed, etc.)

4.4 ReasoningResponse
---------------------
Expected structure of reasoning service responses.

REQUIRED FIELDS:
- success: bool
- result: str (reasoning conclusion)
- confidence: float (0.0 to 1.0)
- reasoning_mode: str (SYMBOLIC, NEURAL, HYBRID)
- execution_time_ms: float

OPTIONAL FIELDS:
- proof_tree: Any (REQUIRED by RedLines.yaml)
- explanation: str
- derived_facts: List[str] (evidence from graph)
- metadata: Dict (must contain agreement_score for HYBRID mode)

5. VALIDATION CHECKS
================================================================================

FortressValidator performs 6 mandatory checks on every response:

5.1 PROOF TREE VALIDATION
--------------------------
CHECK: Does response contain a valid proof_tree?

REQUIREMENTS:
- proof_tree must not be None
- proof_tree.get_proof_depth() >= min_proof_depth (from RedLines.yaml)

VIOLATION TYPE: MISSING_PROOF_TREE
SEVERITY: CRITICAL

CODE LOCATION: _validate_proof_tree()

EXAMPLE FAILURE:
    response.proof_tree = None
    → SecurityBreachException: "Response missing required proof_tree"

5.2 AGREEMENT SCORE VALIDATION
-------------------------------
CHECK: Is agreement_score >= 0.85?

REQUIREMENTS:
- For HYBRID mode: metadata["agreement_score"] must exist
- agreement_score >= 0.85 (RedLines.yaml threshold)
- For SYMBOLIC/NEURAL only: agreement_score not required

VIOLATION TYPE: LOW_AGREEMENT_SCORE
SEVERITY: CRITICAL

CODE LOCATION: _validate_agreement_score()

EXAMPLE FAILURE:
    metadata["agreement_score"] = 0.65
    → SecurityBreachException: "Agreement score 0.65 below threshold 0.85"

CALCULATION (in unified_reasoning_service.py):
    symbolic_set = set(symbolic_response.derived_facts)
    neural_set = set(neural_response.derived_facts)
    overlap = len(symbolic_set & neural_set)
    total = len(symbolic_set | neural_set)
    agreement_score = overlap / total if total > 0 else 1.0

5.3 EVIDENCE LINKAGE VALIDATION
--------------------------------
CHECK: Does response contain graph evidence?

REQUIREMENTS:
- derived_facts list must not be empty
- Indicates reasoning is grounded in knowledge graph

VIOLATION TYPE: MISSING_EVIDENCE
SEVERITY: HIGH

CODE LOCATION: _validate_evidence_linkage()

EXAMPLE FAILURE:
    response.derived_facts = []
    → Violation: "No derived facts found (missing graph evidence)"

5.4 AUDIT TRAIL VALIDATION
---------------------------
CHECK: Are required metadata fields present?

REQUIREMENTS:
- reasoning_mode must be set
- execution_time_ms must be set
- For HYBRID mode: agreement_score must be in metadata

VIOLATION TYPE: AUDIT_TRAIL_INCOMPLETE
SEVERITY: MEDIUM

CODE LOCATION: _validate_audit_trail()

5.5 DETERMINISM VALIDATION
---------------------------
CHECK: Is response deterministic?

REQUIREMENTS:
- NEURAL-only responses without proof_tree flagged as non-deterministic
- Symbolic responses must have proof_tree

VIOLATION TYPE: DETERMINISM_FAILURE
SEVERITY: MEDIUM

CODE LOCATION: _validate_determinism()

5.6 CONTRADICTION DETECTION
----------------------------
CHECK: Are contradictions present?

REQUIREMENTS:
- metadata["contradictions"] must be empty or absent
- Contradictory reasoning is rejected

VIOLATION TYPE: CONTRADICTION_DETECTED
SEVERITY: HIGH

CODE LOCATION: _validate_contradictions()

6. INTEGRATION GUIDE
================================================================================

6.1 DIRECT VALIDATION
----------------------
Use FortressValidator directly for one-off validation:

    from mahoun.core.fortress_validator import FortressValidator
    
    validator = FortressValidator(strict_mode=True)
    result = await validator.validate(response, correlation_id="req-123")
    
    if result.passed:
        print("✓ Response validated")
    else:
        print(f"✗ {len(result.violations)} violations")

6.2 WRAPPED SERVICE (RECOMMENDED)
----------------------------------
Wrap UnifiedReasoningService for automatic validation:

    from mahoun.reasoning.fortress_integration import create_fortress_protected_service
    from mahoun.reasoning.unified_reasoning_service import UnifiedReasoningService
    
    # Create base service
    base_service = UnifiedReasoningService()
    
    # Wrap with Fortress protection
    protected_service = create_fortress_protected_service(
        reasoning_service=base_service,
        strict_mode=True,
        execution_mode=ExecutionMode.DESKTOP_MINIMAL
    )
    
    # Use protected service (all responses auto-validated)
    response = await protected_service.reason(request)

6.3 DECORATOR PATTERN
----------------------
Use decorator for method-level validation:

    from mahoun.reasoning.fortress_integration import fortress_validated
    
    class MyReasoningService:
        @fortress_validated(strict_mode=True)
        async def reason(self, request):
            # ... reasoning logic ...
            return response

6.4 BATCH PROCESSING
--------------------
Validate multiple responses concurrently:

    responses = await protected_service.reason_batch(
        requests=[req1, req2, req3],
        correlation_id_prefix="batch-001"
    )

Track validation metrics:

    stats = validator.get_stats()
